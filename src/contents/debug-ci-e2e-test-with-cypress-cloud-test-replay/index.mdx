## 결론
ci 단계에서 어떻게 e2e 테스트가 돌아갔는지 시각적으로 확인할 수 있는 cypress cloud의 test replay 기능을 통해 e2e 테스트를 좀 더 쉽게 디버깅할 수 있음.

## 서문

e2e 테스트 필요성
우리 회사에 도입
e2e 테스트를 통과하지 못하면 배포되지 않도록 github action을 이용해 구축한 ci(continuous integration) 단계에서도 테스트를 돌리고자 했음

## ci 단계에서 겪은 e2e 테스트의 어려움
cypress에 대한 지식이 적은 상태에서 e2e 테스트를 작성하다보니 테스트가 자주 실패했음. 어쩔 땐 성공하고, 어쩔 땐 실패하는 경우가 있어 원인 파악을 명확히 분별하는 데에 어려움을 겪음.
ci 단계에서의 e2e 테스트는 cypress의 경우 가상 브라우저를 설치하는 과정이 필요함. 여기에 시간이 다소 소모됨.
시각적으로 어떻게 테스트가 돌아가는지 확인하기 어려움.
당시의 네트워크 상태를 알기 어려움.
