짧은 주기로 지속적인 피드백을 나 자신에게 주고자 회고를 매 달 진행해보기로 했다. 그리고 최근 글 쓰기가 뜸해졌다. 이 내용을 쓰기엔 아직 배운 것이 부족하다는 생각때문에 선뜻 글을 작성하기가 망설여졌기 때문이다. 하지만 회고는 근사한 내용이 아니더라도 내가 해왔던 것들을 바탕으로 작성하면 되니 부담이 적겠다 싶었다. (그래도 다음 번엔 개발 관련 내용을 꼭 쓰겠다!)

## 지난 한 달 동안

### Turbo를 이용해 라이브러리 모노레포 만들기(라고 쓰고 toss/slash 따라하기)

우리 회사에서는 2개월 주기로 개인마다 One Goal(하나의 목표)를 정한다. 하고 있는 업무와 연관되어 있어도 되고 그렇지 않아도 된다. 나는 turbo라는 모노레포 형성 도구를 이용해 라이브러리 모노레포를 형성하는 것을 One Goal로 잡았다.

라이브러리 모노레포를 만들고자 하는 이유는 코드 품질 향상을 위해서였다. 몇 개월 전 toss 디자인 시스템 팀에서 업무하시던 프론트엔드 개발자분이 내가 있는 파트로 오시게 되었다. 이 분과 일하면서 인상 깊었던 점은 코드를 선언적으로 잘 작성하신다는 점이었다.

다음은 흔하디 흔한 블로그 같은 웹에서 게시글들을 나열하는 PostList 컴포넌트이다. 특이점은, 각 게시글(li 태그)들마다 구분선인 Divider를 렌더링하고 있고, 마지막 요소는 렌더링을 하지 않는다. 이 특이점을 구현하기 위해 나는 다음과 같이 코드를 작성하곤 한다.

```tsx
export function PostList() {
  const posts = [
    {
      title: '2024년 1월 회고',
      description: '2024년 1월을 회고해봅니다.',
    },
    {
      title: '재활용 가능한 Divider 컴포넌트 만들기',
      description: 'TypeScript와 함께 재활용 가능한 Divider 컴포넌트를 만들어봅니다.',
    },
  ];

  return (
    <ul>
      {posts.map(({ title, description }, index) => (
        <Fragment key={title}>
          <li>
            <h2>{title}</h2>
            <p>{description}</p>
          </li>
          {index !== posts.length - 1 && <Divider />}
        </Fragment>
      ))}
    </ul>
  );
}
```

이 분은 사소하지만 작은 변화를 통해 조금 더 읽기 좋은 코드를 만들어주시곤 한다.

```tsx
// @/utils/array
export function isLastIndex<T>(index: number, array: T[]) {
  return index === array.length - 1;
}

// @/components/post-list
export function PostList() {
  const posts = [
    {
      title: '2024년 1월 회고',
      description: '2024년 1월을 회고해봅니다.',
    },
    {
      title: '재활용 가능한 Divider 컴포넌트 만들기',
      description: 'TypeScript와 함께 재활용 가능한 Divider 컴포넌트를 만들어봅니다.',
    },
  ];

  return (
    <ul>
      {posts.map(({ title, description }, index) => (
        <Fragment key={title}>
          <li>
            <h2>{title}</h2>
            <p>{description}</p>
          </li>
          {!isLastIndex(index, posts) && <Divider />}
        </Fragment>
      ))}
    </ul>
  );
}
```

@/components/post-list에 위치한 PostList 컴포넌트 코드를 볼 때, isLastIndex가 어떻게 구성되어 있는지는 몰라도 함수 이름만으로 어떻게 동작할지 파악이 가능하니 빠르게 코드를 읽어나갈 수 있었다.

사소할 수 있지만 이런 것들이 코드를 조금 더 깔끔하게 만들고 가독성이 좋은 코드 베이스를 만들 수 있겠다는 생각이 들었다. 맡고 있는 프로덕트뿐만 아니라 다른 프로덕트, 그리고 모두 공통적으로 개발하는 백오피스에서도 이를 사용하면 우리 회사 코드 베이스 전체의 품질을 높일 수 있겠다는 생각까지 나아가게 되었다.

그래서 이러한 공통 로직들을 효율적으로 담아낼 수 있는 패키지를 만들고자 했다. 그리고 공통 로직들은 hook이 될 수도 있고 component가 될 수 있기에 여러 패키지가 될 수 있다고 생각했고, 여러 패키지를 효율적으로 관리하려면 monorepo가 적절한 도구가 될 것이라 판단했다.

왜 monorepo인지, monorepo를 위한 도구로 왜 turbo인지, 어떤 monorepo를 참고하며 작업 중인지(소제목에서도 알 수 있지만 toss의 slash를 많이 참고하고 있다)는 따로 글을 작성해보겠다.

### 바빠도 스터디는 참을 수 없다.

작년 12월부터 금요프론트지식탐사대라는 프론트엔드 지식 공유를 위한 사내 모임을 매 주 금요일마다 진행하고 있다. 업무를 하면서 겪은 프론트엔드 버그나 새로 얻은 지식들을 주로 공유하곤 한다. 가끔 feconf나 toss의 slash 영상들, 혹은 인프콘 영상도 시청한다.

나는 이 모임을 진행하면서 종종 취준생 시절에 읽었던 React 공식 문서를 바탕으로 발표를 하곤 했다. React가 왜 strict mode를 만들었는지, useEffect로 data fetching을 하면 왜 race condition이 발생할 수 있는지 등등이었다.

발표를 하고 소감을 말하면서, 나는 물론이고 다들 React를 제대로 이해하고 더 잘 쓰고 싶다는 갈증이 있음을 알게 되었다. 그래서 올해 1월부턴 React 공식 문서 스터디 진행까지 겸하게 되었다.

이렇게 사내에서 프론트엔드 스터디로 총 2개를 진행하고 있다. 이전까지는 사내에 이런 지식 스터디가 없었는데(정확히는 말하자면 사라진 상태였다) 다시 활성화시키고 있다는 점에서 나름 뿌듯함도 느끼고 책임감도 가지게 된다. 그리고 확실히 혼자하기보다는 이렇게 여럿이서 하는 것이 나에게 도움이 됨을 여실히 느낀다. 스터디장인만큼 쉬는 시간을 줄이고 준비해가는 시간을 미리 가져야 하고, 여러 사람에게 지식을 전달하려면 그만큼 더 잘 이해해야 하기 떄문이다. 모르는 것도 스터디 시간에 즉시 공유하면서 공백도 채워나갈 수 있었다. 같이 성장하는 것이 나에게 맞는 방법임을 다시 한 번 느끼게 된다.

### 새로운 집으로

같은 지역에서 같은 지역으로 이사를 간다. 새로운 월세 집을 계약했다. 현재 사는 집은 서울에 상경하게 되면서 조금 급하게 구했었다. 그러다보니 놓친 부분들이 있었다. 일단 조금 좁다보니 점차 답답함이 느껴졌다. 동네는 위치나 인프라 면에서 마음에 들었기에 멀지 않은 곳에서 넓고 더 나은 컨디션의 집을 찾았고 1월 중후반에 새로운 집을 계약했다.

지금 집에서 여러 우여곡절이 많았지만 서울에 적응하게 만들어주고 월세도 많이 아끼게 해준 고마운 집이다. 첫 서울집이라는 점에서 정이 많이 들어 어느 집보다 생각이 많이 날 듯 하다.

## 앞으로

미련한 사람처럼 회상이 많았다. 이제는 어떻게 하면 더 나은 내가 될 수 있을지 새겨보려고 한다.

### 성공을 위해 책 읽기는 선택이 아니라 필수다!

2021년 대한민국 성인들 중 52.5%는 1년 간 책을 1권도 읽지 않았다. 그리고 그 중에 내가 있다.

요즘 개발자로서의 성장과 직장인으로서의 재태크에 관심이 많다보니 유튜브 알고리즘에 관련 영상들이 자주 뜬다. 그러다보면 하나 같이 성공한 사람들은 책을 많이 읽었던 것을 알 수 있게 되었다.

거창할 것이 없이 딱 하루에 1시간... 아니 4... 딱 20분만 책을 읽어보고자 한다! 책이 재밌다면 20분 이상을, 그렇지 않다면 딱 20분만이라도 읽어보자.

책책책! 책을 읽자!

### 선택과 집중

나는 참 욕심이 많은 개발자 같다. 프론트엔드는 가장 잘하고 싶고, 그러면서 백엔드도 할 줄 알며 인프라에 대한 이해도도 가지고 싶다. Korean FE Article, 각종 카카오톡 오픈 채팅방, 서핏 등 여러 군데에서 올라오는 글들을 읽어보고 부족하면 북마크에 저장해둔다. 구매해놓고 미룬 강의를 보면 Docker, NestJS, React Native, 알고리즘, LLM 등 다양하다. 사놓고 제대로 읽지 않은 책도 보면 TypeScript, 네트워크, 프로그래머의 뇌, 리팩터링, 모던 자바스크립트 딥다이브 등이 있다. 부족한 것들 혹은 배우고 싶은 것들을 적어둔 노션 문서를 보면 한 화면을 넘어갈 정도로 가득하다.

이것도 하고 싶고 저것도 하고 싶은 마음에 이도저도 아니게 된다. 이제 조금은 내려놓고, 한 달마다 회고를 하기로 했으니 한 달이라는 시간을 고려해 최대 2개의 주제만 학습하기로 결정했다!

그래서 이번 달에 정한 주제는 우아한 타입스크립트 with 리액트 다 읽기와 개인 블로그 꾸미기이다. 매일 20분 이상 책 읽는 루틴을 통해 우아한 타입스크립트 with 리액트 책을 다 읽어보겠다. 그리고 개인 블로그 꾸미기라고 방대하게 잡았지만 세부적으로는 meta tag 및 sitemap 등 SEO 설정하기, category에 맞는 게시글들 보여주기, 각 게시글마다 thumbnail, supabase를 이용해 조회수 기능 구현하기이다.

앞으로 하고 싶은 것들은 여전히 노션 문서에 저장해놓고 매 달 1일마다 꺼내어 제거하는 식으로 학습해나갈 예정이다.

### 효율적인 휴식

휴식도 똑똑하게 하자. 점심을 먹고 나서 유튜브를 보는 것이 아니라 분리된 공간에서 인생에 도움이 될 만한 책을 읽거나 아예 잠을 자서 에너지를 보충하자.

### 가장 못하는 사람이 되자.

정말 못해서 일을 망쳐버리는 사람이 되자는 것이 아니다. 진심으로 '아직도 난 제일 못하고 팀원들로부터 배울 것이 많은 사람이다.'라는 마음을 되새기자는 것이다.

아직 배워야할 것들이 많다. 이러한 마인드를 커리어 초반에 새겨놓고 겸손함을 잊지 말자.
